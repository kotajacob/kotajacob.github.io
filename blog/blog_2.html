Starting to learn C++

Posted on: 1/20/2016

Over the past few weeks, I have begun a new venture in my programming career in an effort to get back into game development and to start making games that I’m actually proud of. I’m learning C++.

I started by getting the book “Programming Principles and Practice Using C++ 2nd Edition” by Bjarne Stroustrup (the creator of C++). I found it to be extremely well written and full of great examples and challenges. Once I finish the book – which should hopefully happen by the end of this month – I want to move into doing some of the challenges on the /r/dailyprogrammer subreddit. (If you haven’t heard of this sub get go check it out right now – it has hundreds of great little challenges. Also, if you are confused, you can check the comments and see how others were able to solve it so you can get a better idea of where you went wrong.)

I think once I feel comfortable with C++ and can complete most of the easy and medium challenges without too much hair loss I’ll start learning an engine because engines are great and writing big games like I want to make in C++ alone isn’t really practical. The engine I’m looking into right now is Unreal Engine. I’ve downloaded and messed around with it a few times before (and it uses C++ as the scripting language so that’s a huge plus). Part of the reason I’m learning C++ and want to get Unreal instead of say, Unity, the other big competitor I was looking into a bit, is that Unity still doesn’t have a linux engine. I do most of my work on linux boxes these days and I’m not about to use a vm every time I want to do some game design.

- Kota
